## 一、项目概述

FPGA FAE 助手是一个 AI 驱动的现场应用工程师咨询平台，面向 FPGA 开发者提供两大核心服务：

1. **AI 对话 + RAG 文档检索**
   - BYOK（Bring Your Own Key）模式，用户自行配置任意 OpenAI/Anthropic 兼容 API
   - 支持上传 PDF 文档后进行基于文档内容的 RAG 问答
   - 支持豆包多模态 PDF 分析管线（图文识别）

2. **BOM 智能采购模块**
   - 上传/粘贴 BOM 表格（Excel / CSV / PDF / 文本），AI 自动解析元器件信息
   - DeepSeek AI 解析，失败时规则引擎兜底
   - 多平台搜索直通（立创商城、1688、淘宝）
   - 淘宝联盟推广链接（带 PID 佣金追踪）

### 阶段 6：BOM 解析性能大幅优化（2026-02-12 ～ 2026-02-13）

**DeepSeek 解析稳定性问题**

大型 BOM 表（几十行以上）经常触发 DeepSeek 超时，触发规则兜底。

**解决方案演进：**
1. 提高 temperature 到 0.5，简化 prompt（40行→13行）→ 提速但仍超时
2. 调整 timeout 参数，切换 endpoint → 有改善
3. **自适应分块解析**：将大 BOM 拆分为小块并行请求 → 显著提速
4. 触发分块阈值提前（不等超时再切换）→ 减少等待
5. 压缩 chunk 请求 payload，精简字段 → 减少传输时延
6. 加速 chunk 模式 DeepSeek 请求
7. **块级缓存**：相同块内容命中缓存直接返回 → 重复解析极快
8. 解析队列保护（防止并发重复触发）+ 批量写入数据库

**JSON 解析鲁棒性**
- DeepSeek 有时返回 JSON 字符串而非对象 → 兼容 JSON-string 格式
- JSON 截断恢复：不完整 JSON 尝试自动修复
- 规则兜底引擎优化

**其他 BOM 优化**
- BOM 解析支持用户自配 DeepSeek 接口（优先级：用户配置 > 系统环境变量）
- `preferBomKey` 逻辑：BOM 解析优先用 BOM 专用 Key
- 批量删除选中 BOM 条目
- BOM 解析结果本地化（输出中文字段名）

## 八、关键代码模式

### SSE 流式输出

```typescript
const stream = new ReadableStream({
  async start(controller) {
    await aiService.streamChat(messages, (chunk) => {
      controller.enqueue(encoder.encode(`data: ${JSON.stringify({ content: chunk })}\n\n`))
    })
    controller.enqueue(encoder.encode('data: [DONE]\n\n'))
    controller.close()
  }
})
return new Response(stream, {
  headers: { 'Content-Type': 'text/event-stream', 'Cache-Control': 'no-cache' }
})
```
